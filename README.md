# 신사 이벤트 참여봇 V2

## Railway 환경변수
- `BOT_TOKEN`: BotFather에서 발급받은 봇 토큰
- `ADMIN_ID`: 관리자 텔레그램 숫자 ID
- `DB_FILE`: 선택사항. 기본값 `event_bot.db`

## 실행 명령
```bash
python event_bot_v2.py
```

## 관리자 메뉴
텔레그램에서 `/admin`

관리자는 다음 항목을 텔레그램 안에서 변경할 수 있습니다.
- 이벤트 시작/종료
- 이벤트명
- 시작 안내문
- 참여 안내문
- 접수/승인/거절/차단 문구
- 관리자 신청 카드
- 전체 이모지
- 회원 검색
- 승인 대기 및 통계

## 커스텀 이모지
관리자 메뉴 → 이모지 관리에서 다음 형식으로 입력합니다.

```html
<tg-emoji emoji-id="커스텀이모지ID">⭐</tg-emoji>
```

봇이 해당 커스텀 이모지를 사용할 수 있어야 정상 표시됩니다.
지원되지 않거나 접근 권한이 없는 이모지는 일반 대체 문자로 보일 수 있습니다.

## 기존 데이터
기존 `event_bot.db`를 그대로 사용하면 `applications` 데이터는 유지됩니다.
새로운 `settings` 테이블만 자동으로 추가됩니다.

Railway 재배포 후에도 DB를 보존하려면 Volume을 연결하고 `DB_FILE`을
예: `/data/event_bot.db` 로 설정하는 편이 안전합니다.
